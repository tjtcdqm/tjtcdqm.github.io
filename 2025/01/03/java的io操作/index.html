Java的io操作

初学Java，总是会对Java中的多而杂的io操作忌惮三分。本文不是对所有的io操作类进行讲解，而是尝试梳理JavaIO的基本思路，帮助理解Java中的io数据流思想，同时能将流这样的思想用在理解其他语言的流操作上。

流

什么是流？为什么Java要设计流？它的出现能帮助程序员更好地理解程序的io操作吗？这些疑问是我在看到Java的io操作时的真实疑问。毕竟在C语言中只要printf和scanf就可以读写了，加上read,write写文件，再不济，就加上sprtinf,sscanf,fprinf,fscanf就解决了，这些函数的语义都很明确，利于理解。其实在当你在使用fopen,open函数时，使用二进制格式打开一个文件时，你就是在把他当作一个二进制字节流来处理。

任何数据都被视为一组序列，这样的序列就是数据流。这个流可以是你的键盘输入（可以被抽象为一组有序的数据），一个文件（在存储设备中存储的有序的数据），网络（其他计算机上的存储设备的有序数据）。Java把他们都看成一种东西，即一种数据的流。

其实稍微了解Linux系统的同学就会知道，Linux中也对文件的定义做了扩展，即只要是数据流就被视为文件，就可以像操作文件一样对这个数据流操作。但是这样的理解在Windows中是不存在的。

而在Java中，为了忽略底层的操作系统，忽略各种硬件，对读写的操作都视为流。也就是说，不管你是Linux还是Windows还是mac，不管操作系统对文件的理解是不是数据流，我Java都只对数据流进行读写，你底层都要为我Java实现这个抽象。这样，Java就统一了读写的接口。

怎么操作这些数据流

怎么理解流这个抽象呢？为什么要把这个抽象叫做流？
这个流(stream)可以理解为溪流，是一个名词，但是溪流本身是一直处于运动中的，即溪流可以从一个地方流到另一个地方

Java中，把数据都看作这样的流，这些流就像水一样，可以从一个地方（文件1）移动到另一个地方（文件2）。这样的流动就是io！

为了操作这样的流，我们就需要一条管道，这个管道就是Java中的各种IOstream。一个管道有两个端口，在创建一个管道的时候，要指定管道的一个端口，这个端口就对应一个数据流。

read和write函数可以看为一个抽水泵的作用，让数据以流的形式，从管道中通过，流向另外的一个流中。

在Java中这样类型的管道很多，虽然他们的名字都带有stream，但是他们并不是流。他们只是能容许流通过的一条管道。并且需要指定两端的真正的流才能有意义的工作。

字符流和字节流

使用不同类型的管道，已经能很好地处理这些流了，但是我们还是遇到了新的问题。那就是文本文件。
stream管道会把所有的数据流都视为字节流，即数据都是以8位来流动的。比如FileInputStream和一个read方法，就会返回接收的一个字节。

如果都是acsii字符还好，如果遇到中文，这样的读取就会没有意义。说到底，就是字节流这个抽象还是太过于底层了，我们需要一个字符流的管道，管道中的数据不以字节为单位流动，而以字符为单位流动。(千万不要认为字符就是单字节，所以字节流等于字符流。一个字符不一定是单字节组成的，比如中文字符)

reader 和 writer

reader和writer是就是这样的字符流管道，他们是对数据更进一步的抽象，它代表了特定种类的流。

InputStreamReader 类作为 Reader 的一个具体实现类，它封装了 InputStream，并将字节流转换为了字符流。

reader作为管道，也需要两个端口才能进行有意义的工作。只不过它的一端是一个字节流管道。

在reader中的read方法，就不再是读取一个字节，而是读取一个字符了。

在reader中更是实现了readline这样的函数，按行读取字符串，这是一般的stream做不到的.

BufferedReader

如果使用reader读文件，那么每次读取都会使用一次系统调用，大量地读取会非常耗时。这是为了提升性能，我们就使用了bufferedReader。他能一次将很多内容加载到内存中，之后读取就会在内存中进行，不需要额外的系统调用，节省了大量时间。但是有可能导致缓冲区溢出的情况。

什么是System.in 和System.out

他们是java.lang.System 类中的预定义静态成员，分别代表标准输入流和标准输出流。

System.in： 这是 InputStream 类的一个实例，通常表示标准输入流。在Java程序运行时，默认情况下它连接到用户的键盘输入。通过 System.in 可以读取从键盘输入的数据，例如使用 Scanner 类或者 BufferedReader 类等工具来读取用户输入的文本信息。

System.out： 这是 PrintStream 类的一个实例，通常表示标准输出流。在Java程序运行时，默认情况下它连接到控制台输出。通过 System.out 可以向控制台输出文本信息，这是最常见的输出信息的方式，使用 println、print 等方法。
